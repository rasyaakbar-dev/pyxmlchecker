# Security Policy

## Supported Versions

Currently, only the latest version of `pyxmlcheck` is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability within `pyxmlcheck`, please do not open a public issue. Instead, report it privately to ensure the safety of our users.

### How to Report

Please send an email to **<rasyaakbarnugraha@gmail.com>** with the following information:

- A description of the vulnerability.
- Steps to reproduce the issue.
- Potential impact of the vulnerability.
- (Optional) Any suggested fixes or mitigations.

### Our Response

1. **Acknowledgment**: We will acknowledge receipt of your report within 48 hours.
2. **Investigation**: We will investigate the issue and determine its severity.
3. **Remediation**: If a vulnerability is confirmed, we will work on a fix.
4. **Disclosure**: Once a fix is released, we will coordinate a public disclosure if necessary.

Thank you for helping keep `pyxmlcheck` secure!
